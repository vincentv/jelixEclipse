jDao : mapping objet relationnel

jDao permet de générer des objets PHP qui opèrent sur des tables précises de base de données. La génération de ces objets inclus la génération des requêtes SQL qui correspondent à ces opérations.

Pour ce faire, jDao se base sur un fichier XML que vous devez fournir. Ce fichier permet de s’affranchir de l’écriture fastidieuse des requêtes SQL, permet de tenir compte des problématiques du genre SQL injection : les objets générés s’occupent de tout. Le fichier XML peut être généré par un outil de développement, comme par exemple l’une des commandes fournies avec le script Jelix.

À partir d’un fichier XML de ce type, jDAO fourni donc deux objets, un objet “record” et un objet “factory”, selon le pattern DAO.

Un objet record représente un enregistrement : ses propriétés correspondent avec les champs d’une ou plusieurs tables.

Un objet factory fourni un certain nombre de méthodes permettant de créer un record, de le sauvegarder, de le détruire ou de récupérer des ensembles d’objets record.
