Créer et gérer de formulaire avec jForms

ATTENTION : jForms, tels qu’il est décrit dans ces pages, n’est disponible qu’à partir de Jelix 1.0 beta3, ou en test dans les version nightlies actuelles de Jelix. L’implémentation de jForms dans Jelix 1.0 beta2 n’est pas aussi complète, totalement obsolète et ne sera pas documentée.

À l’avenir, jForms pourra aussi générer d’autres types de formulaires (en ajax, XUL, XForms etc..) sans changer une seule ligne de code, si ce n’est le plugin de template qui est chargé de l’afficher.

Créer un formulaire

jForms se base sur des fichiers descriptifs en XML. Tout formulaire est décrit en XML, avec une grammaire assez simple, à mi-chemin entre HTML et XForms.

Ces fichiers XML sont à placer dans un répertoire forms des modules et doivent avoir comme nom foo.form.xml, où foo est le nom identifiant du formulaire (que vous choisissez bien sûr). Cet identifiant est utilisé alors dans les selecteurs.

Par exemple “monModule~produit” fait référence au formulaire monModule/forms/produit.form.xml.

Pour un déscriptif des balises à utiliser dans un fichier jforms, voir la page correspondante.

Un fichier jForms xml est compilé par jForms en une classe PHP qui est mise en cache à la première utilisation. L’objet correspondant à un formulaire hérite de jFormsBase.
